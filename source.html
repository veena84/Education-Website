How to disable browser back button using JavaScript

This is a pure JavaScript solution "How to disable browser back button using JavaScript".If you want todisable browser back button for particular page then put below sourcecode in that particular and and if you want to disable browser back on all pages then put below sourcecode in your master page.I would not recommend you to disable your browser back button if you want to make your website user friendly.
As I said above I would not recommend you to disable your browser back button if you want make your site user friendly but if you want to implement then you do not have to do anything but just copy the sourcecode and paste in a page where you want to disable your browser back button and if you want to disable browser back button on all pages then you can put it in your master page of your website.

Disable browser back button with JavaScript

Sourcecode:

<script type="text/javascript> 
history.pushState(null, null, location.href);
    window.onpopstate = function () {
        history.go(1);
    };
</script>

Here if you read the title then you will know that this is regarding all the checkboxes of a particular row only not all the checkboxes of all rows and columns.So if you consider just one row then in first row there will be many columns.Let's take an example which is shown below.You have 10 columns in a row.Now whichever checkbox you select in that row, all the checkboxes after that selected checkbox in a row will be selected using jQuery.

How to create your first Laravel Project

We are going to continue from where we left last.After installing Fresh Laravel project on your PC , go command prompt and go to the path where your project is located and inside your project folder give below command.

php artisan make:auth

 e.g. C:\xampp\htdocs\myapp>php artisan make:auth

 Note: Here "myapp" is my project name

We are creating our project with authentication so Laravel will handle all the requests further.It will be easy for us to check user through authentication rather than manually.

Above command will install a layout view, registration and login views, as well as routes for all authentication end-points.A HomeController will also be generated to handle post-login requests to your application's dashboard.

Below files added or updated in your project through above command:

1) HomeController.php (Controller File)
2) web.php (Route File)
3) home.blade.php (View file)
4) auth and layouts folder in views

Now We need to create tables in our database.If you have not created your database then create one database after that go to ".env" file in your Laravel project.In this ".env" file set "DB_DATABASE", "DB_USERNAME" and "DB_PASSWORD".

DB_DATABASE = "Your database name that you want to use for this project"
DB_USERNAME =  "Username for your database"

Now let's create database tables through migration in Laravel.Take a look at below easy steps to create database tables in your Laravel project.

Steps to Create tables through Migration in Laravel


Step 1) Create migration file through command prompt
Step 2) Now go to database folder and inside that folder one migration folder will be there and you will see one file there which you just created.Now in that file, you will see two functions.In up function you have to write code to create project.And in down function you have to write opposite code to your up function.In shorts, in this step you will write code in up and down function.
Step 3) Now you have to run this migration file in your command prompt so that up function will be executed.And if you rollback migration then down function will be executed.That's it.

If you are a beginner then you might not understand above steps but don't worry and start doing what I do and you will understand it in a better way.Let's see above steps.

Example

 Step 1) To create migration file, write below line in your command prompt.Here "php artisan make:migration" is command which you will use every time to create any migration file and "create_first_table" is migration file name.You can give any name as your migration file name.Don't forget to open command prompt in our project location.

 php artisan make:migration create_first_table

 Step 2) In this step, we will add code to up function to create table and in down function we will add code which is opposite to to the code in up function.So here if up function is creating table then we will add code to drop table in our down function.Why we are writing opposite code in down function because up() and down() will work as do and undo.

If you made any mistake while creating table and if you want to make changes in that then you can rollback your migration which will drop table in our case then make changes to your up function then run migration again then again table will be created wit changes you made.

Code for up() function in your migration file

 Schema::create('users', function (Blueprint $table) {
$table->increments('id');
$table->string('name');
$table->string('email');
$table->string('password');
$table->timestamps();
});

Explanation: first line will create "users" table.In that id,name,email and password column will be there.Here "increments" will be integer datatype and auto increment and "string" will be varchar datatype.Here we also added "timestamps" which will add two more columns "created_at" and "updated_at".If you see in your database table then one more column will be added which is "remember_token".

Code for down() function in your migration file

Schema::drop('users');

Explanation: this line simply drops the table.

Step 3) In your command prompt, execute above up() function by writing below code.Below code will migrate the migration file.

 php artisan migrate

 To execute down function, write below code in your command prompt.Below code will rollback the migratation.

php artisan migrate:rollback

Now open your project in your browser and you will see login and register at right top corner after we use "php artisan make:auth" because when we installed Laravel and opened project in our browser, it was not there but now you are able to see login and register.

How to use Nayjest DataGrids with Laravel

In our last post we saw jQuery DataTables Plug-in which provides multi-column sorting, Pagination and Instant search but if there are suppose 10,000 records and we are showing 10 records per page in that table then this Plug-in loads all 10,000 records first and then it shows 10 records per page so in loading that page it takes too much time and this is not good at all from user point of view so we are going to see "Nayjest DataGrids" which does the same work and we are going to use it with Laravel.So how to integrate "Nayjest DataGrids" with Laravel ? Just follow below steps use "Nayjest DataGrids" with Laravel.

In my earlier post I saw youto set up your Laravel project on Windows.If you have completed that then you can follow these steps otherwise I will suggest you to setup Laravel on your PC/Laptop first.

Steps to use Nayjest DataGrids with Laravel

Step 1) In your composer.json file add below code in "require"

"require": {
        "nayjest/grids": "^1.3.1",
        "laravelcollective/html": "^5",
        "nayjest/laravel-doctrine-dbal": "^1.0"
},
Step 2) Add below code in your "providers" of app.php file of config folder

'providers' => [
        Nayjest\Grids\ServiceProvider::class,
        Collective\Html\HtmlServiceProvider::class,
        Nayjest\LaravelDoctrineDBAL\ServiceProvider::class,
],
Step 3) Add below code in your "aliases" of app.php file of config folder

'aliases'=>[
         'Form'  => Collective\Html\FormFacade::class,
        'HTML'  => Collective\Html\HtmlFacade::class,
        'Grids'     => Nayjest\Grids\Grids::class,
        'Input' => Illuminate\Support\Facades\Input::class,
        'DBAL' => Nayjest\LaravelDoctrineDBAL\Facade\DBAL::class,
],
Step 4) go to your Laravel project folder then press (Shift + right click) and click on "open command window here" and in command window write "composer update" and hit enter.In short you have to update your composer, that's it.

You have now integrated Nayjest DataGrids with Laravel Successfully.Now let's see the example.

Create one route like shown below.You have to give thsis path to your route file which will be located in routes folder and name will be "web.php"

web.php(Route File)

Route::get('table', 'tablecontroller@showtable');
now whenever you write table after pubic in your url it will call showtable method of your tablecontroller.

tablecontroller.php(Controller File)

go to your project path inside of your project folder, press (shift + right click) choose "open command window here" and write below line to make controller through artisan

 php artisan make:controller tablecontroller
Now open your editor and in that open your "tablecontroller.php" file that you just created and paste below code in that file.Your tablecontroller will look like below.
<?php
namespace App\Http\Controllers;
use Illuminate\Http\Request;
use DB;
use Grids;
use App\User;
use Nayjest\Grids\DbalDataProvider;
use Nayjest\Grids\FieldConfig;
use Nayjest\Grids\GridConfig;
use Nayjest\Grids\Grid;
use Nayjest\Grids\FilterConfig;

class tablecontroller extends Controller
{
    public function showtable(){
    $query = DB::getDoctrineConnection()->createQueryBuilder();
        $query
            ->select([
                'id',
                'firstname',
                'email',
                'address',
                'gender',
                'interest',
                'mobile'
            ])
            ->from('data');
        $cfg = (new GridConfig())
            ->setDataProvider(
                new DbalDataProvider($query)
            )
            ->setPageSize(8)
            ->setColumns([
(new FieldConfig('id'))->setSortable(true)->addFilter((new FilterConfig)->setOperator(FilterConfig::OPERATOR_LIKE)),
                (new FieldConfig('first_name'))->setSortable(true)->addFilter((new FilterConfig)->setOperator(FilterConfig::OPERATOR_LIKE)),
                (new FieldConfig('last_name'))->setSortable(true)->addFilter((new FilterConfig)->setOperator(FilterConfig::OPERATOR_LIKE)),
                (new FieldConfig('address'))->setSortable(true)->addFilter((new FilterConfig)->setOperator(FilterConfig::OPERATOR_LIKE)),
                (new FieldConfig('phone_number'))->setSortable(true)->addFilter((new FilterConfig)->setOperator(FilterConfig::OPERATOR_LIKE)),    
(new FieldConfig('interest'))->setSortable(true)->addFilter((new FilterConfig)->setOperator(FilterConfig::OPERATOR_LIKE)),
                (new FieldConfig('mobile'))->setLabel('Phone Number')->setSortable(true)->addFilter((new FilterConfig)->setOperator(FilterConfig::OPERATOR_LIKE)),            ]);
        $grid = new Grid($cfg);
       /* $grid = Grids::make($cfg);*/
        return view('table', compact('grid'));
    }

}

How to use jQuery DataTables Plug-in

This is another good feature of jQuery that we can use in our projects where we have large number of data to display.Suppose we have 10,000 users registered and they are stored in our database table.Now you want to fetch all users and want to show them on single page.You know it very well how to get all users from database and show them to on html or user's side.

If you are using PHP, CodeIgniter, Laravel or Wordpress or any other CMS or Framework, you will fetch all records from database table and then you will run a for loop and you will print them all in table format.But it does not look good that you print all users in just one page.I am working on Laravel so I know that we can give pagination before we call to any view page but paginate will give only pagination not sorting or filter(search) options in that.

So to use these all functionality together and also showing just limited records on one page we can use jQuery DataTables Plug-in.Now let's understand jQuery DataTables Plug-in.

You can also refer "How to use Nayjest DataGrids with Laravel" which does the same functionality.For more kind of different solution, you can check it out more examples here -> jQuery DataTables Examples

What is jQuery DataTables Plug-in?

DataTables is a Plug-in for the jQuery JavaScript library.It is a highly flexible tool, based upon the foundations of progressive enhancement, and will add advanced integration controls like Pagination, Instant Search, and multi-column ordering to any HTML table.

In short, one page will contain our both page, that page we will use to add the user and also to edit the user.All the things we are doing earlier for edit page we will do the same.suppose you have created one user and that user is stored in database table and now you are on dashboard where all users will be displayed.

How to do Soft Deletes in Laravel

If you want to delete any record temporary but not permanently in Laravel then here is the solution for that.If you are working on Laravel then try to create files through migration only if possible.It will help you a lot ahead.

Now let's guess your going to add one migration file where you will create one new table and for records in that table, you want to add Soft Deletes functionality which will not delete records permanently but temporary.So let's see what Soft Deletes functionality does.

What Soft Deletes Functionality does in Laravel ?


Whenever you add Soft Deletes functionality in any table of your database, it creates one column "deleted_at" in that table.By default its value will be null.After adding this Soft Deletes functionality in your table, if you delete any record from controller then it will be deleted temporary it means for that record which you deleted through controller, its "deleted_at" column value will be set with current time.So you will have idea that if "deleted_at" column has any value for any record  then that record is temporary deleted.So whenever you fetch data from that table you have to make sure that you fetch only that records whose "deleted_at" column has null value.

Steps to add Soft Deletes functionality to any table of your database


Step 1) while creating new table, you add below line so it will also create column "deleted_at" in your table.If you want to add this functionality in your existing table then also you can do it in same way.You have to create new migration file and in that file you have to add below line.

$table->softDeletes();

Step 2) Save and Run the migration.

Step 3) Open model of that table and add below code in that table.Also make "deleted_at" column fillable.

Outside of class, add below line,

use Illuminate\Database\Eloquent\SoftDeletes;

Inside class, add below lines,

use SoftDeletes;   

protected $dates = ['deleted_at'];

Now whenever you delete any record in controller, you have to delete that record same way you were earlier doing and deleted_at column value will be set with current time for that record.It will not be permanently deleted.

Whenever you want you can restore that record using restore() method.
If you want to permanently delete then you have to you forceDelete() method.
